# Налаштування відправки листів на Gmail

## Як це працює

Форми відправляють листи **напряму на ваш Gmail** через Next.js API route. Не потрібні сторонні сервіси типу Formspree або EmailJS.

## Крок 1: Створення App Password в Gmail

1. Перейдіть на [myaccount.google.com](https://myaccount.google.com)
2. Увійдіть у свій Gmail акаунт (`agrolan.seed.drills@gmail.com`)
3. Перейдіть до **Безпека** (Security)
4. Увімкніть **Двофакторну аутентифікацію** (якщо ще не увімкнено)
5. Прокрутіть вниз до **Паролі додатків** (App passwords)
6. Натисніть **Створити** (Generate)
7. Оберіть **Пошта** та **Інший (назва користувача)**
8. Введіть назву: "SVR1 Landing Page"
9. Натисніть **Створити**
10. Скопіюйте згенерований пароль (16 символів)

## Крок 2: Налаштування змінних середовища

Створіть файл `.env.local` в корені проєкту (якщо його ще немає) та додайте:

```env
GMAIL_USER=agrolan.seed.drills@gmail.com
GMAIL_APP_PASSWORD=ваш_16_символьний_пароль_з_кроку_1
```

**Важливо:** 
- Не додавайте `.env.local` до Git (він вже в `.gitignore`)
- Після додавання змінних перезапустіть сервер: `npm run dev`

## Крок 3: Перевірка роботи

1. Заповніть форму на сайті
2. Натисніть "ВІДПРАВИТИ ЗАЯВКУ"
3. Перевірте ваш Gmail - має прийти лист з даними форми

## Якщо не працює

1. Перевірте, чи правильно скопійовано App Password (16 символів)
2. Переконайтеся, що двофакторна аутентифікація увімкнена
3. Перезапустіть сервер після додавання змінних середовища
4. Перевірте консоль браузера на наявність помилок

## Безпека

- App Password безпечніший за звичайний пароль
- Можна видалити App Password в будь-який момент
- Листи надсилаються тільки на ваш Gmail
- Немає залежності від сторонніх сервісів

## Для продакшену

На хостингу (Vercel, Netlify тощо) додайте змінні середовища в налаштуваннях проєкту:
- `GMAIL_USER` = ваш Gmail
- `GMAIL_APP_PASSWORD` = ваш App Password
